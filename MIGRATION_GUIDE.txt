KEY CHANGES:
------------

1. PACKAGE.JSON:
   - Backup current package.json: mv package.json package.json.backup
   - Use new package.json: mv package.json.nuxt package.json
   - Install dependencies: npm install

2. EMAIL FUNCTIONALITY:
   - Contact form now uses /api/send-mail endpoint
   - Server-side email sending with nodemailer
   - Configuration via environment variables

3. ENVIRONMENT SETUP:
   - Copy .env.example to .env
   - Configure SMTP settings:
     * SMTP_HOST: Your SMTP server (e.g., smtp.gmail.com)
     * SMTP_PORT: SMTP port (587 for TLS, 465 for SSL)
     * SMTP_USER: Your authenticated domain email
     * SMTP_PASS: Your email password/app password
     * CONTACT_RECEIVER: Email where contact form submissions will be sent

4. AUTO-IMPORTS:
   - Components in components/ are auto-imported
   - Composables in composables/ are auto-imported
   - Stores in stores/ are auto-imported
   - No need for manual imports

INSTALLATION & SETUP:
----------------------

1. Backup current package.json:
   $ mv package.json package.json.backup

2. Use Nuxt package.json:
   $ mv package.json.nuxt package.json

3. Install dependencies:
   $ npm install

4. Configure environment variables:
   $ cp .env.example .env
   $ nano .env  # Edit with your SMTP settings

5. Run development server:
   $ npm run dev

6. Build for production:
   $ npm run build

7. Preview production build:
   $ npm run preview

SMTP CONFIGURATION:
-------------------

For Gmail:
- SMTP_HOST=smtp.gmail.com
- SMTP_PORT=587
- SMTP_USER=your-email@gmail.com
- SMTP_PASS=your-app-password (generate at: https://myaccount.google.com/apppasswords)

For Custom Domain:
- SMTP_HOST=mail.yourdomain.com
- SMTP_PORT=587
- SMTP_USER=noreply@yourdomain.com
- SMTP_PASS=your-password

EMAIL FLOW:
-----------
FROM: SMTP_USER (your authenticated email)
TO: CONTACT_RECEIVER (your admin email)
REPLY-TO: Client's email (from form)
SUBJECT: "New Contact: [Client Name]"

This ensures:
- Emails are sent from your authenticated domain
- You receive submissions at your admin email
- You can reply directly to the client

API ENDPOINT:
-------------
POST /api/send-mail

Body:
{
  "name": "string",
  "email": "string",
  "subject": "string",
  "message": "string"
}

Response (Success):
{
  "success": true,
  "message": "Email envoyé avec succès"
}

Response (Error):
{
  "statusCode": 400/500,
  "statusMessage": "Error message"
}
